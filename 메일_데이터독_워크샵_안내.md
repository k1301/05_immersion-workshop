Bedrock 기반 기업용 AI Agent 워크샵의 Datadog LLM Observability 실습 파트 관련하여 환경 구성 및 시나리오를 공유드립니다.

---

## 워크샵 개요

LangGraph + Amazon Bedrock 기반의 사내 업무 에이전트를 구축하고, Datadog LLM Observability로 에이전트의 동작을 모니터링/진단하는 실습입니다.

에이전트 주요 기능:
- 사내 문서 검색 (RAG, Bedrock Knowledge Base)
- IT 헬프데스크 티켓 관리 (AgentCore Gateway → MCP)
- 외부 검색 (Google Search)

아키텍처:
```
사용자 → Chainlit UI → LangGraph + Bedrock Claude
                              ↓
                      Tool Calling (ReAct)
              ┌──────────┼──────────┐
           search_kb   Gateway    google_search
              │        MCP tools
         Bedrock KB      │
                   CloudFront → ALB → ECS
                   (IT Helpdesk REST API)
```

## 환경 구성 방법

사전 준비:
- AWS 계정 (Bedrock 모델 액세스 활성화, 리전: us-east-1)
- AWS CLI 설정 완료
- Docker Desktop 설치 및 실행
- Datadog 계정 + API Key

```bash
# 1. 리포지토리 클론
git clone <REPO_URL>
cd 05_immersion-workshop

# 2. 전체 인프라 배포 (VPC, ECR, ALB, ECS, KB 등 한 번에)
./deploy-master.sh

# 3. AgentCore Gateway 생성 (AWS 콘솔에서 수동)
#    - Helpdesk CloudFront URL + openapi.json 등록
#    - Gateway MCP URL 확인

# 4. Gateway MCP URL 반영하여 재배포
./deploy-master.sh
#    → Gateway MCP URL 입력

# 5. Chainlit UI 접속하여 동작 확인
#    → Output의 AgentcoreCloudFrontURL
```

deploy-master.sh가 자동으로 처리하는 항목:
- VPC, 서브넷, IGW, 라우팅 생성
- ECR 리포지토리 생성
- Docker 이미지 빌드(linux/amd64) 및 ECR 푸시
- ALB, ECS Cluster/Service, Bedrock Knowledge Base, S3 생성
- KB 문서 S3 업로드 및 데이터 소스 동기화

## Datadog LLM Observability 연동

```bash
cd agentcore-backend
./deploy-datadog.sh
```

입력 항목:
- Datadog API Key
- Datadog Site (기본: datadoghq.com)
- ML App 이름 (기본: agentcore-backend)

연동 확인: Chainlit UI에서 질문 입력 → 1~2분 후 Datadog 콘솔 → LLM Observability → Traces에서 트레이스 확인

## Datadog 실습 시나리오

오류 시나리오는 에이전트에 내장되어 있어 별도 환경변수 변경 없이, 질문 내용에 따라 자동으로 정상/오류가 분기됩니다.

### 시나리오 1: 토큰 에러 진단

"요약", "정리", "상세히", "자세히" 등의 키워드가 포함된 질문을 하면 자동으로 max_tokens=10이 적용되어 토큰 에러가 발생합니다.

Chainlit UI에서 질문:
```
"휴가 정책을 상세히 요약 정리해줘"     → 에러 ❌ (토큰 초과)
"연차 휴가는 며칠까지 쓸 수 있나요?"   → 정상 ✅
```

Datadog에서 확인:
- Traces → 에러 트레이스 클릭
- LLM span에서 stop_reason: max_tokens 확인
- 정상 트레이스와 비교하여 max_tokens 값 차이 확인

### 시나리오 2: Failure to Answer 진단

보안 관련 키워드(보안, VPN, 비밀번호 등)가 포함된 질문을 하면 자동으로 잘못된 KB ID로 검색합니다.

Chainlit UI에서 테스트:
```
"휴가 신청 방법 알려줘"     → 정상 ✅
"경비 처리 절차 알려줘"     → 정상 ✅
"IT 보안 정책 알려줘"       → 에러 ❌
"VPN 접속 방법 알려줘"      → 에러 ❌
"비밀번호 변경 방법 알려줘"  → 에러 ❌
```

Datadog에서 확인:
- 에러 트레이스 vs 정상 트레이스 비교
- search_kb span에서 ResourceNotFoundException 확인
- 보안 키워드 질문에서만 잘못된 KB ID 사용하는 패턴 발견

### 시나리오 3: 모델별 퍼포먼스 비교

```bash
DD_LLMOBS_ENABLED=1 \
DD_LLMOBS_ML_APP=agentcore-benchmark \
DD_LLMOBS_AGENTLESS_ENABLED=1 \
DD_API_KEY=<API_KEY> \
DD_SITE=<DD_SITE> \
ddtrace-run python benchmark.py --models sonnet-4.5,haiku-3.5,sonnet-3.7
```

- 7개 질문 × N개 모델 자동 실행
- Datadog LLM Observability에서 모델별 레이턴시, 토큰 사용량, 응답 품질 비교
