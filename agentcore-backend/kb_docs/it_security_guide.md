# IT 보안 가이드

## 1. 비밀번호 정책

### 1.1 비밀번호 요구사항
- 최소 12자 이상
- 대문자, 소문자, 숫자, 특수문자 포함
- 90일마다 변경 필수
- 최근 5개 비밀번호 재사용 불가

### 1.2 다중 인증(MFA)
- 모든 임직원 필수 활성화
- Google Authenticator 또는 Duo 사용
- VPN 접속 시 MFA 필수

## 2. 데이터 보안

### 2.1 데이터 분류
- **공개(Public)**: 외부 공개 가능
- **내부(Internal)**: 사내 직원만 접근
- **기밀(Confidential)**: 승인된 직원만 접근
- **극비(Restricted)**: 임원 및 특정 직원만 접근

### 2.2 데이터 처리 규칙
- 기밀 이상 데이터는 암호화 필수
- 개인정보는 GDPR/개인정보보호법 준수
- 고객 데이터는 프로덕션 환경에서만 접근

### 2.3 이메일 보안
- 의심스러운 첨부파일 열지 않기
- 피싱 이메일 의심 시 security@company.com 신고
- 기밀 문서 전송 시 암호화 필수

## 3. 디바이스 보안

### 3.1 회사 노트북
- FileVault(Mac) 또는 BitLocker(Windows) 암호화 필수
- 화면 잠금: 5분 이내 자동 설정
- 최신 보안 패치 자동 업데이트

### 3.2 모바일 디바이스
- 회사 이메일 접속 시 MDM 등록 필수
- 분실 시 즉시 IT 헬프데스크에 신고
- 원격 초기화 기능 활성화

### 3.3 USB 및 외부 저장장치
- 승인된 암호화 USB만 사용 가능
- 미승인 저장장치 사용 금지
- IT 팀에 USB 승인 요청

## 4. 네트워크 보안

### 4.1 VPN 사용
- 재택근무 시 VPN 필수 사용
- 공용 Wi-Fi 사용 시 VPN 연결
- VPN 클라이언트: Cisco AnyConnect

### 4.2 Wi-Fi 정책
- 회사 Wi-Fi: `CompanySecure` (WPA3)
- 게스트 Wi-Fi: `CompanyGuest` (인터넷만 가능)
- 개인 핫스팟 사용 시 VPN 필수

## 5. 소프트웨어 설치

### 5.1 승인된 소프트웨어
- IT 포털에서 Self-Service 설치 가능
- 미승인 소프트웨어 설치 금지
- 오픈소스 라이브러리는 보안 스캔 후 사용

### 5.2 라이선스 준수
- 정품 소프트웨어만 사용
- 개인 라이선스 업무 사용 금지

## 6. 보안 사고 대응

### 6.1 보안 사고 발견 시
1. 즉시 작업 중단
2. IT 헬프데스크에 긴급 티켓 생성
3. Security 팀에 이메일 전송: security@company.com
4. 감염된 디바이스는 네트워크에서 분리

### 6.2 사고 유형
- 악성코드 감염
- 랜섬웨어 공격
- 피싱 이메일 클릭
- 데이터 유출
- 계정 해킹

## 7. 교육 및 인증

- 신입 직원: 입사 1주일 내 보안 교육 이수
- 전 직원: 연 1회 보안 교육 필수

### 7.2 보안 인증
